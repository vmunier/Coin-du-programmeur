Raccourcis Eclipse

Le fichier contenant les définitions de vos raccourcis Eclipse s'appelle org.eclipse.ui.workbench.prefs et devrait se trouver dans workspace/.metadata/.plugins/org.eclipse.core.runtime/.settings/

Pour utiliser les raccourcis d'un autre programmeur, il suffit de remplacer votre fichier org.eclipse.ui.workbench.prefs par le sien.

mon fichier de raccourcis Eclipse

J'utilise le plugin Emacs+ sous Eclipse afin de profiter de nombreux raccourcis Emacs que j'appréci particulièrement pour faire de "l'édition brut" :
les opérations comme supprimer une ligne, se déplacer de mots en mots font parti de "l'édition brut" tandis que lancer tous les tests junit ou trouver un fichier dans le workspace n'en font pas parti.

Pour l'édition brut, nous avons donc les raccourcis d'emacs.
Pour tout le reste, nous gardons les raccourcis intégrés nativement dans eclipse.

Vous trouverez ci-dessous une présentation des raccourcis que je trouve les plus utiles.
La grande majorité sont les raccourcis définis par défaut dans Eclipse.

Tous les raccourcis écrits en rouge sont ceux que j'ai redéfinis personnelement ou faisant parti d'Emacs+ .
Il ne font donc pas parti des raccourcis natif d'Eclipse.
Pour chaque raccourci redéfini, j'indiquerai le raccourci d'origine pour exécuter la même commande.

Les plus connus

  • Ctrl+1 quick-fix / spell-checker
  • Ctrl+Shift+O organise les imports : supprimme les imports inutilisés, ajoutte les manquants
  • Alt+/ complète le mot
    • appuyez plusieurs fois d'affilé sur Alt+/ pour accèder aux autres complètions
  • Ctrl+Shift+F formate/indente le fichier source
  • Alt+? (Ctrl+Espace par défaut) autocompletion
    • Le fameux Ctrl+Espace que j'ai redéfini ^^ car je l'utilise déjà pour avoir le même fonctionnement que sous Emacs, c'est à dire commencer une marque.

Navigation

  • Alt+Gauche revient en arrière dans l'historique de navigation (je l'utilise pour revenir à l'endroit où j'ai fait un Ctrl+J)
  • Alt+Droite pour naviguer en avant (l'inverse de alt+gauche)
  • Alt+Haut/Bas déplace la ligne courante (ou plusieurs séléctionnées) en haut ou en bas
  • Ctrl+/ commente ou décommente la séléction ou la ligne courante
  • Ctrl+. positionne le curseur sur la prochaine erreur ou avertissement
  • Ctrl+, pour l'erreur précédente
  • Ctrl+Shift+P saute jusqu'à l'accolade/parenthèse correspondante
  • Ctrl+Alt+H identifie tous les endroits où est appelée la méthode dans laquelle on se trouve
  • Ctrl+! (Ctrl+Alt+Shift+L par défaut) change de vue
    Étant sous linux, ce raccourcis par défaut me posait problème car Ctrl+Alt+L (qui est une suite très proche) verrouille la session.
  • Ctrl+J (F3 par défaut) J pour Jump, saute à la séléction ou la déclaration se trouvant sous le curseur

Refactoring

Fonctionne dans l'éditeur et dans le package explorer :
  • Alt+Shift+L extrait le texte séléctioné et le place dans une variable locale
    (ça détermine et écrit le type pour vous)(Local)
  • Alt+Shift+M extrait le texte séléctioné et le place dans une méthode (Method)
  • Alt+Shift+I inline la méthode séléctionnée (Inline)
  • Alt+Shift+R renomme la séléction ou le nom sous le curseur (Rename)
  • Alt+Shift+V déplace la méthode ou le champ (moVe)
  • Alt+Shift+T accès au menu de refactoring (refacToring)

Exécution

(eXecute)
  • Alt+Shift+X T exécute les tests untaires du fichier courant
  • Alt+Shift+X J exécute le main du fichier courant
  • Alt+Shift+X R exécute sur le serveur

Il y en a d'autres. Les options vous sont indiquées après avoir tapé Alt+Shift+X.
Alt+Shift+X peut être remplacé par Alt+Shift+D dans tous les exemples du dessus pour éxécuter en mode debug.

Ouverture

Cherche dans tous les projets ouvert :
  • Ctrl+Shift+T ouvre le fichier contenant la classe dont on précise le nom
  • Ctrl+X Ctrl+F (Ctrl+Shift+R par défaut) ouvre n'importe quel type de fichier en lui précisant son nom
    Je n'ai pas gardé le raccourci par défaut car Ctrl+X Ctrl+F est un raccourcis Emacs auquel je suis habitué.

Les correspondances sont trouvées sur le début du nom de classe/fichier.
Vous pouvez commencer votre recherche avec un astérisque * pour rechercher n'importe classe/fichier qui contient ce mot.

Génération

  • Alt+Shift+J ajoute un commentaire javadoc
  • Alt+S R génère les getters et setters
  • Alt+S V insère les signatures de méthode pour les méthodes surchargées depuis la superclass ou l'interface

Menu

  • Alt+Shift+N affiche la même chose que lorsque vous utilisez la souris et cliquez sur File->New
  • Alt+N ouvre l'assistant pour créer une classe Java
    Ensuite, maintenir Alt et pour chaque description de champ, une lettre est soulignée par un underscore _ .
    C'est un indice pour taper le raccourci correspondant.
    Par exemple le 'e' de Browse... est souligné lorsqu'on appui sur Alt donc Alt+E sert à ajouter une superclass.
    Alt+A pour une interface, le 'a' de Add... est souligné lorsqu'on appui sur Alt etc...
Utile pour naviguer dans un menu, comme celui des getters et setters par exemple :
  • Shift+Gauche/Droite replie/déplie une liste
  • Espace séléctionne, coche un choix
