soit par obligation (université, entreprise), soit par intérêt personnel. Je cherche toujours les meilleurs outils pour programmer un projet personnel et être le plus productif possible. Le langage de programmation est le plus important de ces outils. Dans l'ordre de mon apprentissage : _ le C Super langage. Super rapide. _ l'ActionScript 3 (ou AS3) car je voulais (et je veux encore) créer des jeux sur internet. Flash dominait ce marché à l'époque. Il permettait de jouer directement dans son navigateur. _ VB.NET (par obligation, pour mon premier travail en entreprise) : J'ai découvert que le choix du langage dans lequel on programme est important. C'est super chiant de programmer en VB.NET . _ Java Apris à l'université. Très bon langage. Peu de liberté offerte au programmeur. _ Python Apris à l'université. J'aime lire et écrire du code Python car je le trouve concis et beau. Concis car Python offre de nombreux mécanismes de haut niveau pour écrire ses programmes. Beau car le code Python est épuré de parenthèses et accolades qu'on retrouve en masse dans les langages de la famille C.
Je préfère les langages compilés et statiquement typés aux langages interprétés et dynamiquement typés. En effet, le compilateur peut m'indiquer beaucoup d'erreurs que j'aurai pu ne pas remarquer tout seul. _ Scala Par intérêt personnel. J'entendais beaucoup de bien sur Scala. Scala est très puissant. _ Haskell Par intérêt personnel. Je me suis intéressé à Haskell pour apprendre en profondeur les concepts de la programmation fonctionnel que j'avais commencé à voir avec Scala. Je suis content d'avoir utilisé Scala avant Haskell car ça m'a permis d'aborder Haskell en douceur. J'aurai eu beaucoup de mal à apprendre Haskell si ça avait été mon premier langage fonctionnel.

langages classés par ordre de préférence, le premier étant celui que je préfère. courte description avec des points :

C

J'ai découvert la programmation avec ce langage.
Le C est un langage de "moyen niveau". Il est définitivement plus haut niveau que l'assembleur et offre plusieurs abstractions. Pourtant, une bonne maitrise du C requiert de bien connaitre la gestion de la mémoire d'un ordinateur. La gestion des pointeurs n'est pas facile à assimiler. Le programmeur C sait ce qu'il fait
Je trouve que coder dans ce langage est fun ! J'aime sa rapidité d'exécution. Comprendre les pointeurs et la gestion de la mémoire nous permet de mieux comprendre le fonctionnement de langages de plus haut niveau. En Java par exemple, on comprend tout de suite ce qu'est une référence ou encore ce qu'il se passe lorsqu'on fait un 'new'. La liberté d'écriture qu'offre ce langage. qui le rend dangereux Avantages et Inconvéniants Très peu de concept : +facilite la maitrise du langage -ne facilite pas la vie du programmeur.

Java

J'aime
  • la grande quantité de bibliothèques disponibles
  • les IDE Java (Eclipse est mon préféré) qui sont super avancés pour ce langage
  • la portabilité du langage; en comparaison le C est portable mais vous avez intéret à respecter la norme ANSI si vous souhaiter que votre programme C s'exécute partout pareil.
Je trouve que coder dans ce langage est chiant. Il n'offre aucune liberté d'écriture. L'avantage est d'avoir des programmes Java qui, grossièrement, se ressemblent tous. Mais en tant que programmeur, ça limite mon plaisir de coder en Java.

Python

ce que j'aime dans le langage J'aime lire du code python.
Le code Python est souvent très concis. Il n'y a pas de déclaration de types.
J'aime aussi l'indentation qui délimite les blocs d'instructions et qui, à mon sens, rend le code plus lisible car il est épuré des différents caractères ( ) { } ; qu'on retrouve dans les langages de la famille C.
